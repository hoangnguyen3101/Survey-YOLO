# YOLO Multi-Model Training Workspace

This project is set up to facilitate parallel training and inference of YOLO models on the BDD100K dataset, optimized for a single-GPU environment with Weights & Biases tracking.

## 📂 Project Structure

- **`modules/`**: Contains core Python scripts (`train.py`, `infer.py`, `valed.py`).
- **`data/`**: Dataset configurations (`bdd100k.yaml`) and the BDD100K-10K dataset.
- **`models/`**: (Optional) Custom YOLO architecture YAML files.
- **`weights/`**: Store your pre-trained weights (`.pt` files) here.
- **`runs/`**: Output directory for training experiments (logs, weights, plots).
- **`inference/`**: Output directory for prediction results.
- **`utils/`**: Helper scripts for data processing.

## 🚀 Getting Started

### 1. Environment Setup
Activate your conda environment:
```bash
conda activate yolo
```
### 3. Running Training

#### Option A: Automated Parallel Training
To train specific models:
```bash
./train_parallel.sh yolov3.pt or yolov5s.pt or yolov8s.pt or yolov10s.pt
```
*If no arguments are provided, it trains the default set (v3, v5s, v8s, v10s).*

#### Option B: Manual Training (Recommended for Log Monitoring)
Open separate terminal tabs (or `tmux` panes) and run:
- **v3**: `python modules/train.py --model yolov3.pt --name yolov3_exp`
- **v5s**: `python modules/train.py --model yolov5s.pt --name yolov5s_exp`
- **v8s**: `python modules/train.py --model yolov8s.pt --name yolov8s_exp`
- **v10s**: `python modules/train.py --model yolov10s.pt --name yolov10s_exp`

> [!WARNING]
> **GPU VRAM Limit**: Since you have 1 GPU, running all 4 models simultaneously might cause an `Out of Memory` (OOM) error. Monitor your VRAM usage with `nvtop`.

### 4. Running Inference
To run inference across multiple weights:
```bash
./infer_parallel.sh
```
